Once NodeJS is installed, you can use npm or npx or vite or next.js to create a React application;

Using CRA(Create React App)

  • npx create-react-app my-app
  • cd my-app
  • npm start - Runs the development server;

Using Set React App

Using Vite

  • npm create vite@latest my-app --> Choose React or React + TypeScript when prompted;
  • cd my-app
  • npm install
  • npm run build → bundles the app for production;
  • npm run dev
    • It starts the vite's development server, which serves your app locally (at http://localhost:5173 for Vite or http://localhost:3000 for CRA)
    • It watches your files for changes and automatically reloads the browser(hot-reloading);
  • npm start - could mean development or production, depends on config;

Using Next.js - You can also use Next.js for creating a React application;

  • Check package.json of existing applicaiton, to find if it is using npx or vite or next.js;

React Project Structure

my-app/ │ ├── node_modules/ # Installed dependencies; 'npm install' downloads all these dependencies; No need to commit this heavy folder (auto-generated); ├── public/ # Static files (served as-is, e.g. favicon, images) │ └── index.html # Root HTML file, React mounts into here │ ├── src/ # Source code of your app │ ├── assets/ # Images, fonts, global styles, etc. │ ├── components/ # Reusable UI components (buttons, navbars, etc.) │ ├── pages/ # Page-level components (Home, About, Dashboard, etc.) │ ├── layouts/ # Layout wrappers (e.g. sidebar, header, footer) │ ├── hooks/ # Custom React hooks (e.g. useAuth, useFetch) │ ├── context/ # React Context providers (e.g. AuthContext) │ ├── services/ # API calls, external integrations │ ├── utils/ # Helper functions, constants │ ├── App.jsx # Main app component │ ├── main.jsx # Entry point, renders into index.html │ └── index.css # Global styles │ ├── package.json # Project metadata(name, description) + dependencies ├── package-lock.json # Exact dependency versions ├── .gitignore # Files ignored by Git(e.g. node_modules) └── README.md # Project documentation
