What is JSX(JavaScript XML)?
  • Component is JavaScript function that take in props as input and returns JSX;
  • It is not HTML, but it lets you write HTML inside JavaScript code;
  • It is more convinent way of creating HTML elements, than React.createElement() calls;
  • Eg: const element = <h1>Hello, world!</h1>; Behind the scenes, react uses tools like Babel to convert JSX into JS code: const element = React.createElement("h1", null, "Hello, world!");
Comments
  • Comments can be added within curly braces inside JSX and won’t be included in the rendered output;
  • Eg: {/* This is a comment */}
Self closing tags
  • JSX support self-closing tags, similar to HTML;
  • It’s useful for elements without content, such as images or icons;
  • Eg: const element = <img src="logo.png" />;
Expression
  • Inside the curly brackets, we can write an expression that evaluates to some value;
    • const MyComponent = () => { const name = "John"; return ( <div> <p>Hello, {name}!</p> </div> ); }
Attributes
  • JSX attributes look like HTML attribute, but use camelCase for most;
  • class → className, for → htmlFor; Eg: const element = <img src="logo.png" className="app-logo" />;
Conditional Rendering
  • Expressions are used to conditionally render elements;
    • const App = () => { const number = 10; return ( <div> {number > 0 ? ( <p>Number {number} is positive</p> ) : ( <p>Number {number} is Negative</p> )} </div> ); };
Fragment
  • A component can't return multiple elements; Rather, wrap them in div-like element OR fragment;
    • // ❌ Invalid return ( <h1>Title</h1> <p>Paragraph</p> ); // ✅ Valid return ( <div> <h1>Title</h1> <p>Paragraph</p> </div> ); // ✅ Valid - Fragments return ( <div> <h1>Title</h1> <p>Paragraph</p> </div> );
Event Handlers
  • React event handling is very similar to handling DOM events in plain JavaScript, but with a few important differences;
  • CamelCase naming // ✅ Correct <button onClick={handleClick}>Click me</button> // ❌ Incorrect <button onclick={handleClick}>Click me</button>
  • Pass a function, not a string
    // ✅ Correct <button onClick={handleClick}>Click me</button> // ❌ Incorrect <button onClick="handleClick">Click me</button>
  • Inline event Handlers, Eg: <button onClick={() => alert("Hello!")}>Click Me</button>
  • React passes a synthetic event object (wrapper around native events) to the handler: function App() { function handleClick(event) { console.log(event); // SyntheticBaseEvent console.log(event.type); // "click" } return ( <button onClick={handleClick}>Click me</button> ); }
  • You can pass arguments to event handlers; function App() { function greet(name) { alert(`Hello, ${name}!`); } return ( <button onClick={() => greet("Alex")}>Greet</button> ); }
