What is a Hook?
  • Hooks are used to maintain state and side effects, in functional components;
  • Before hooks (introduced in React 16.8), developers mainly used class components to handle state and lifecycle logic(do something when component loads or updates);
  • Hooks allows us to same thing using functional components, making code simpler and more reusable;
Challenges with Class Components
  • Multiple lifecylce methods
    • Related logic gets split across multiple lifecycle methods;
    • Beginners must understand the order of lifecycle methods;
    • A data fetch might involve: componentDidMount (fetch start), componentDidUpdate (refetch when props change), componentWillUnmount (cleanup);
    • useHook allows us to handle all of this logic in one place;
  • Difficult to reuse logic
    • If you want to share code that deals with state between components, you usually need complex patterns; These can make your components harder to understand;
  • Confusing 'this' binding
    • this doesn’t work automatically in class methods;
    • You have to bind methods manually in the constructor or use arrow functions;
Rules of Hooks
  • name of a Hook must start with use;
  • Only call hooks at the top level (not inside loops, conditions, or nested functions);
  • Only call hooks from React functions (functional components or custom hooks);
Types of Components
  • Function based components - we can use simple function(rfc) or arrow function(rafc);
  • Class based components - we are not using them after v16;

import React, { useState } from "react"; function Counter() { // Declare a state variable "count" with initial value 0 const [count, setCount] = useState(0); return ( <div style={{ textAlign: "center", marginTop: "20px" }}> <h2>Counter: {count}</h2> <button onClick={() => setCount(count + 1)}>Increase</button> <button onClick={() => setCount(count - 1)}>Decrease</button> <button onClick={() => setCount(0)}>Reset</button> </div> ); } export default Counter; import React from "react"; import Counter from "./Counter"; function App() { return ( <div> <h1>React State Example</h1> <Counter /> </div> ); } export default App;
  • useState(0) → creates a state variable count with initial value 0.
  • setCount → function to update the state.
  • Every time setCount is called, React re-renders the component with the new state value.
  • The buttons call setCount with new values.
  • Higher-Order Components - A higher-order component is a function that takes a component and returns a new component. HOCs are used to enhance components with additional functionality. For eg, for Implement cross-cutting concerns like authentication or logging.
  • Pure Components - Pure components are a type of class component that optimizes performance by implementing a shallow comparison in shouldComponentUpdate. Useful for improving performance in large applications.
  • Controlled Components: Manage their state through React. Example: Form elements with value controlled by state.
  • Uncontrolled Components: Manage their state through the DOM. Example: Using ref for accessing DOM elements directly.
