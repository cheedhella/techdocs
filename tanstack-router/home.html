Types of Routing
  • Code based routing
    • Routes are defined programatically using functions like createRoute, createRootRoute;
  • File based routing
    • Routes are generated automatically, based on file structure and specific naming conventions;
    • For example, about.jsx in a pages folder becomes the /about route;
    • Here are the rules:
      • Root route - Use __root.tsx in the root directory for the app's root route;
      • Dot Separator - denotes nested root; For eg, blog.post.tsx becomes /blog/post
      • Dynamic segments - Add $ for parameterized routes; For eg, blog.$id.tsx for /blog/:id
      • Layouts - Prefix files or folders with _ to create layouts not matched directly in URLs;
      • Index Routes - Use index for exact parent path matching; dashboard.index.tsx for /dashboard
      • Lazy Loading - Use .lazy.tsx to enable code splitting;
      • Reserved routes - Routes starting with /api are reserved for API usage. Adjust via the apiBase option if needed;
      • Ignored Files - Files/folders prefixed with - are ignored by default, configurable in tsr.config.json;
API Explaination
  • createRootRoute
    • It defines entry point of your applicaiton;
    • You should pass it to the RouterProvider;
    • It returns a RootRoute instance that has methods like: addChildren(), useLoaderData() etc;
    • You need to nest all other routes under it;
    • Config:
      • path -> Path is implicit for root route;
      • component -> react component to render;
      • loader -> data fetching logic;
      • errorComponent -> error boundary for that route;
      • pendingComponent -> loading state handler;
    • API:
  • createFileRoute
    • Use it for file based routing, never mix file-based routing with manual addChildren();
  • Route, RootRoute, FileRoute, Link
    • getParentRoute -> returns parent root(for children, not needed for root);
    • addChildren -> to add child routes;
    • useLoaderData -> to access data fetched by loader;
    • useRouteError -> to access error thrown by loader or action;
    • useMatch -> to get match object for the route;
    • useParams -> to get params for the route;
    • useSearch -> to get search params for the route;
    • useNavigate -> to navigate programmatically;
    • useLocation -> to get current location object;
    • useIsPending -> to check if a navigation is in progress;
    • useIsRouting -> to check if the router is currently routing;
    • useRouter -> to get the router instance;
  • Outlet
    • It is a placeholder component;
    • It tells the router: when a child render it here inside the parentâ€™s layout; Otherwise, nested routes would never appear;
  • Link
    • It is a component to create links;
    • It accepts to prop for the target route;
    • It handles client-side navigation;

    • Error Handling
